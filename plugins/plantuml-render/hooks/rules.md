## PlantUML 图形绘制规则（ECA 插件自动注入）

本规则由 plantuml-render 插件在每次 LLM 请求前自动注入。

### 画图规则

当需要表达架构、流程、数据流、组件关系等视觉化内容时，**必须**使用 PlantUML 语法输出 `@startuml...@enduml` 代码块（禁止 ASCII 手绘、Mermaid.js 等替代方案）。

### 语法速查

| 场景 | 语法 |
|------|------|
| 简单箭头 | `[A] --> [B] : 描述` |
| 时序图 | `A -> B : 消息` |
| 组件图 | `component [名称]` |
| 包图 | `folder "名称" { }` |
| 类图 | `class 名称 { }` |

完整语法参考：https://plantuml.com/zh/

### 禁止行为（零容忍）

| 禁止 | 示例 |
|------|------|
| ASCII 箭头链 | `A --> B --> C` |
| ASCII 框图 | `┌──┐` 等 box-drawing 字符 |
| 文本拼接图 | `+---+` 等用 `+` `-` `|` 拼接 |
| Mermaid.js | 用 mermaid 替代 PlantUML |

### 语法自检清单

输出 `@startuml...@enduml` 代码块前，检查以下项：

1. `@startuml` 和 `@enduml` 必须成对出现
2. 箭头语法：`-->`（实线）、`..>`（虚线）、`->`（时序图），不要混用不存在的箭头
3. 每条语句独立成行，不要在行尾堆叠多条语句
4. 包含特殊字符（`[`、`]`、`{`、`}`、`"`）时注意成对闭合
5. 方括号 `[组件名]` 中的组件名不要含 `[` 或 `]` 自身

### 输出后处理

输出 `@startuml...@enduml` 代码块后，**调用 `render_plantuml` 工具**渲染为图片。
工具接受 `source` 参数（完整的 PlantUML 源码），返回 PNG 图片。
图片由客户端自动渲染展示，无需在回复中手动嵌入 data URI。
