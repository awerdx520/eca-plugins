#+TITLE: ECA PlantUML 渲染插件
#+AUTHOR: ECA Plugin System
#+DATE: 2025-07-19

* 概述

  这是一个 ECA (Emacs Code Assistant) 插件，通过 MCP 服务器提供 =render_plantuml=
  工具，将 PlantUML 源码渲染为 PNG 图片（base64 内联），在客户端直接显示。

  采用 *二层架构* ：

  1. =L1 preRequest Hook= — 每次 LLM 请求前自动注入画图规则（PlantUML 语法、禁止行为、自检清单）
  2. =MCP 服务器= — 提供 =render_plantuml= 工具，模型输出 =@startuml= 块后主动调用工具渲染

* 工作原理

  PlantUML 源码 → pre-request hook 注入画图规则 → LLM 输出 @startuml 块
  → LLM 调用 render_plantuml 工具 → MCP 服务器执行 plantuml -tpng
  → base64 PNG data URI 返回给 LLM → 嵌入回复展示。

  采用 *二层架构* ：

  1. =L1 preRequest Hook= — 每次 LLM 请求前注入画图规则和语法自检清单
  2. =MCP 服务器= — 提供 =render_plantuml= 工具，模型主动调用完成渲染

* 安装

  在 ECA 配置文件（~/.config/eca/config.json）中添加插件源和安装声明：

  #+BEGIN_SRC json
  {
    "plugins": {
      "github-eca-plugins": {
        "source": "https://github.com/your-org/eca-plugins.git"
      },
      "install": ["plantuml-render"]
    }
  }
  #+END_SRC

  或者本地开发/测试方式：直接将本目录作为插件源：

  #+BEGIN_SRC json
  {
    "plugins": {
      "local-dev": {
        "source": "/home/thomas/Workspace/eca-plantuml-plugin"
      },
      "install": ["plantuml-render"]
    }
  }
  #+END_SRC

  重启 ECA 后生效。

* 依赖

  - ~plantuml~ CLI（需要 Java 运行环境）
  - ~python3~（Hook 脚本 JSON 处理）

* 配置

  暂无外部配置项。所有行为由 Hook 脚本控制。

* 故障排查

  | 问题 | 原因 | 解决 |
  |------|------|------|
  | PlantUML 代码块未渲染为图片 | render_plantuml 工具未被调用 | 检查 package.json 是否正确，查看 ECA 日志中 MCP 服务器是否启动成功 |
  | 图片显示为空白 | PlantUML 语法错误 | 检查源码中的 PlantUML 语法，确保 ~@startuml~ 和 ~@enduml~ 正确 |
  | MCP 服务器启动失败 | Node.js 依赖未安装 | 在插件目录执行 ~npm install~ |
  | plantuml 命令不可用 | 未安装 PlantUML | 安装 ~plantuml~ （需要 Java 运行环境） |
